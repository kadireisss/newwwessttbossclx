## BOSS Cloaker v3.2 — Comprehensive Feature Enhancement & Stability Upgrade

### Performance
- **Domain slug cache** (30s TTL) — eliminates per-request DB queries on `/r/:slug`
- **Stats query optimization** — 4 separate `COUNT(*)` queries → 1 with `FILTER`
- **Settings cache** (60s TTL) — frequently accessed config cached in memory

### New Features
- **Country detection** — extracts country from `CF-IPCountry` / `X-Vercel-IP-Country` headers and logs it
- **Country allow/block filtering** — `allowedCountries` / `blockedCountries` fields now enforced in cloaker engine
- **Query parameter forwarding** — UTM params, `gclid`, `fbclid`, `msclkid` etc. forwarded to target URL on redirect
- **Landing page CSS/JS injection** — `cssContent` and `jsContent` fields now injected into landing page HTML
- **Domain status check** — `paused` → show landing page, `maintenance` → show maintenance page
- **Auto-blacklist** — bots scoring ≥90 automatically added to IP blacklist when `auto_block` setting is enabled
- **Dashboard stats** — API now returns `botPercentage`, `todayVisits`, `todayBots`, `todayReal`

### Stability
- **Route error handling** — all CRUD routes wrapped in try/catch with user-friendly Zod validation messages
- **Duplicate blacklist protection** — IP and UA blacklist entries checked before insert
- **Landing page deletion safety** — returns 409 Conflict if domains reference the page
- **WebSocket broadcast log noise** — removed per-broadcast `console.log`
- **Version badge** updated to v3.2

### Files Changed
- `server/storage.ts` — slug cache, settings cache, stats optimization, duplicate protection, deletion safety
- `server/routes.ts` — country detection, query forwarding, CSS/JS injection, status check, auto-blacklist, error handling
- `server/websocket.ts` — removed broadcast logging
- `shared/routes.ts` — expanded dashboard stats Zod schema
- `client/src/pages/dashboard.tsx` — use API-provided stats, version bump
